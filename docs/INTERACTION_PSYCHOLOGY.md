# Психология общения AthleteCore

Как система поддерживает спортсменку без давления и с согласием (HITL) перед разбором ошибок.

## Принципы

1. **Сначала человек, потом тактика** — после поражения или плохого настроения не вываливать список ошибок.
2. **Без стыда** — запрещён тон «ты не умеешь», давление, сравнение с другими.
3. **Согласие на разбор** — технический debrief только после «да» / «давай разберём» (human-in-the-loop в диалоге).
4. **Победа** — сначала искренне отметить успех, потом опционально предложить углубиться.
5. **Не только «мягкий» коуч** — часть спортсменов хотят **жёсткий, чёткий** разбор; система не «потакает» всем подряд, а уважает стиль из памяти и явный запрос.

## Режимы (`interaction_mode`)

| Режим | Когда | Поведение AI |
|--------|--------|----------------|
| `support_first` | проигрыш, грусть, «всё плохо», много ошибок в негативном тоне | Поддержка, 1 микро-шаг, **без JSON ошибок** |
| `celebrate_first` | победа, «хорошо сыграла» | Похвала, **без «но надо улучшить»** |
| `full_analysis` | явное «да» на предложение разбора ИЛИ прямой запрос разбора | Обычный Analyst + JSON карточка |
| `neutral` | нейтральный спортивный вопрос | Стандартный анализ |

## Тон разбора (`coaching_tone`)

| Тон | Когда | Поведение |
|-----|--------|-----------|
| `gentle` | по умолчанию; согласие «да» без жёстких слов | Чёткий разбор, без стыда, умеренно тепло |
| `direct` | в памяти `interaction.support.style: direct` или «чётко укажи ошибки» | Нумерованный список ошибок, без «воды» |
| `tough` | в памяти `style: tough` или «жёстко / твёрдо» | Прямо и жёстко по фактам, без оскорблений |

**Сразу полный разбор** (без support_first), если в том же сообщении явный запрос:
«разбери мои ошибки», «нужно чёткое понимание», «жёстко укажи» — даже после поражения.

## Двухходовый сценарий (HITL в чате)

```text
Спортсменка: «Проиграла, куча ошибок на задней линии, устала»
  → support_first, в конце: «Хочешь разберём ошибки? (да/нет)»
  → в память: interaction.pending_offer = analysis_debrief

Спортсменка: «да»
  → full_analysis, разбор + Analysis card
  → pending_offer сбрасывается
```

## Настройки (сейчас и после регистрации)

Ключи в LTM (процедурная память):

- `interaction.support.style` — gentle | direct | tough | motivational
- `interaction.support.when_low` — как поддерживать после поражения
- `interaction.support.when_high` — как реагировать на победу
- `interaction.pending_offer` — ожидает ли система ответа да/нет

**Сейчас:** можно записать вручную через чат («поддерживай мягко после поражений») — extraction подхватит.

**Позже (регистрация):** короткий опрос 3–4 вопроса → те же ключи в LTM.

## Код

- `backend/app/graph/interaction.py` — тон, согласие, режим
- `backend/app/graph/prompts.py` — системные промпты
- `backend/app/graph/nodes.py` — маршрутизация Analyst/Health
- `backend/app/graph/runner.py` — checkpoint `offer_followup` + память

## Тест в чате

1. «Проиграла сегодня, расстроена, много ошибок» → поддержка, без карточки ошибок.
2. «да» → разбор с карточкой.
3. «Выиграла финал!» → празднование, предложение углубиться.
